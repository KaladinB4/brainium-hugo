# llms.txt & Sitemap Strategy for Brainium

## Executive Summary

**Recommendation:** Implement both llms.txt (static) and leverage Hugo's built-in sitemap.xml generation. Consider adding sitemap.txt as well for human-readable version.

**Why:** llms.txt helps AI systems understand your content structure and provides context about the site's purpose. It complements sitemap.xml which is machine-focused for search engines.

---

## What Are These Files?

### 1. llms.txt (Emerging Standard)
- **Purpose:** Provide LLM-friendly context about your website
- **Location:** `/llms.txt` (root of site)
- **Format:** Markdown with specific structure
- **Use case:** Helps AI assistants understand your content when users ask questions
- **Status:** Proposed standard by Jeremy Howard/Answer.AI (gaining adoption)

**Key Benefits:**
- Provides context window for LLMs (solves "too much content" problem)
- Can include summaries and descriptions of your content
- Helps AI answer questions about your site accurately
- Human-readable format

**Format Structure:**
```markdown
# Site Name

> Brief summary/context about the site

Optional details paragraphs

## Section Name
- [Link Title](URL): Description
- [Another Link](URL): Description

## Optional
- [Secondary content](URL): For shorter context windows
```

### 2. sitemap.xml (Established Standard)
- **Purpose:** List all pages for search engine crawlers
- **Location:** `/sitemap.xml` (root of site)
- **Format:** XML
- **Use case:** SEO, search indexing
- **Status:** Google/Bing standard for 20+ years

**Hugo:** Automatically generated by default

### 3. sitemap.txt (Human Alternative)
- **Purpose:** Plain text list of URLs (human-readable sitemap)
- **Location:** `/sitemap.txt` (optional)
- **Format:** Plain text
- **Use case:** Quick reference, debugging
- **Status:** Alternative to XML sitemap

### 4. robots.txt (Existing)
- **Purpose:** Guide web crawlers on what to access
- **Location:** `/robots.txt` (root of site)
- **Format:** Plain text directives
- **Status:** Already implemented ✓

## Options Analysis for Brainium

### Option A: llms.txt Only (Static File)
**Pros:**
- Simple implementation - just create one file
- Full control over what AI sees
- Curated content list (not everything)
- Can add context/summaries
- Fast for AI to parse

**Cons:**
- Manual maintenance (must update when adding content)
- May miss new content if not updated
- No automation

**Best for:** Sites with stable content, curated approach, want full control

**Implementation:** Create `/static/llms.txt` manually

---

### Option B: llms.txt + sitemap.txt (Both Static)
**Pros:**
- llms.txt for AI context
- sitemap.txt for human-readable URL list
- Both simple to maintain
- Complementary purposes

**Cons:**
- Two files to maintain
- Still manual

**Best for:** Want both AI context and human reference

**Implementation:** Create both `/static/llms.txt` and `/static/sitemap.txt`

---

### Option C: Dynamic llms.txt (Hugo Template)
**Pros:**
- Auto-generated from content
- Always up-to-date
- Can use front matter fields
- Scalable

**Cons:**
- More complex to set up
- May include too much (llms.txt should be curated)
- Need template logic

**Best for:** Large sites, frequently updated content, want automation

**Implementation:** Create `/layouts/llms.txt` template

---

### Option D: Hugo's Built-in sitemap.xml + Static llms.txt
**Pros:**
- sitemap.xml auto-generated (already happening)
- llms.txt provides curated context
- Best of both worlds
- sitemap.xml for search engines
- llms.txt for AI understanding

**Cons:**
- Still need to maintain llms.txt manually

**Best for:** Most sites, including Brainium

**Implementation:** 
1. Verify Hugo generates sitemap.xml (add to config if not)
2. Create `/static/llms.txt` with curated content

---

### Option E: Full Suite (llms.txt, sitemap.xml, sitemap.txt, robots.txt)
**Pros:**
- Complete coverage
- SEO optimized (sitemap.xml)
- AI optimized (llms.txt)
- Human reference (sitemap.txt)
- Crawler guidance (robots.txt)

**Cons:**
- More files to manage
- Potential redundancy

**Best for:** Enterprise sites, maximum discoverability

---

## My Recommendation for Brainium

**Choose Option D: Hugo sitemap.xml + Static llms.txt**

**Rationale:**
1. **sitemap.xml** - Hugo already generates this (verify it's working)
2. **llms.txt** - Create manually to provide curated context about Brainium's mission and key content
3. **Brainium's content is high-quality and curated** - manual llms.txt fits the editorial approach
4. **Content doesn't change daily** - manual maintenance is manageable
5. **Provides context** - llms.txt can explain the "why" behind the research, not just list URLs

## Implementation Plan

### Step 1: Verify sitemap.xml Generation

Check hugo.toml for:
```toml
[outputs]
  home = ['HTML', 'RSS']
  section = ['HTML', 'RSS']
  # sitemap.xml should be auto-generated
```

Or add explicit sitemap config:
```toml
[sitemap]
  changefreq = 'monthly'
  filename = 'sitemap.xml'
  priority = 0.5
```

### Step 2: Create llms.txt

Create `/static/llms.txt`:

```markdown

Key principle: Use AI to externalize effort, not responsibility for thinking. We focus on Mode 3 (Amplification)—you think first, AI sharpens.

- [Microsoft Critical Thinking Survey](/research/microsoft-critical-thinking-2025/): 319 knowledge workers, 40% of AI tasks involved no critical thinking—driven by confidence mismatch

## Practical Frameworks

- [Confidence Calibration Protocol](/frameworks/confidence-calibration-protocol/): Four-tactic system to maintain critical thinking through confidence tracking
- [Active Prompting Protocol](/cognitive-tools/ai-assisted-learning/): Tactical approach for coding without skill degradation

## Field Validation

- [2 Weeks Confidence Calibration](/field-notes/confidence-calibration-experiment/): 47 AI tasks logged, 3 errors caught, confidence gap narrowed over time

## Philosophy

- [Core Philosophy](/about/philosophy/): Three Modes framework—Replacement (bad), Convenience (risky), Amplification (optimal)
- [Cognitive Offloading Research](/essays/cognitive-offloading-research/): 40 years of research on tools that boost performance but degrade skills

## Optional

- [Content Marketing Funnel](/docs/content-funnel.md): How TOFU → MOFU → BOFU → Retention content connects
```

## Comparison: llms.txt vs sitemap.xml

| Feature | llms.txt | sitemap.xml |
|---------|----------|-------------|
| **Format** | Markdown | XML |
| **Purpose** | AI context & understanding | Search indexing |
| **Content** | Curated, with descriptions | All pages, technical metadata |
| **Size** | Small (context window friendly) | Can be large |
| **Maintenance** | Manual (recommended) | Auto-generated |
| **Usage** | LLM inference time | SEO crawling |
| **Includes summaries?** | Yes | No |
| **External links?** | Yes | No |

## Advanced Options (Future)

### llms-ctx.txt (Expanded Context)
- Automatically expanded version with full content of linked pages
- Created using `llms_txt2ctx` CLI tool
- For when you want to provide full documentation context

### Markdown Versions of Pages
- Proposed standard: same URL + `.md` extension
- Example: `/research/mit-cognitive-debt-2025.md`
- Hugo can generate these with alternate output formats
- Requires template work

### Section-level llms.txt
- `/docs/llms.txt` for documentation section
- `/research/llms.txt` for research articles
- More granular context
